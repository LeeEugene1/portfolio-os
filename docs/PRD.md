# 제품 요구사항 정의서

## 목표

OS 느낌의 반응형 포트폴리오 웹앱을 만들고, 배포된 웹앱을 Flutter WebView APK로 패키징합니다.

이 제품은 일반적인 스크롤형 포트폴리오 사이트가 아니라, 데스크톱 OS처럼 앱 아이콘을 열어 프로그램 창을 사용하는 포트폴리오 경험을 제공합니다. 웹앱이 제품의 본체이며, Flutter 앱은 동일한 웹앱을 모바일 앱 형태로 제공하는 셸입니다.

## 컨셉

웹앱은 가벼운 데스크톱 OS처럼 느껴져야 합니다. 사용자는 바탕화면에 배치된 앱 아이콘을 클릭하거나 탭해서 Portfolio, Store, Resume, Contact, Calculator 프로그램 창을 엽니다.

PC 브라우저, 모바일 브라우저, Flutter WebView 안에서 같은 제품 모델을 유지하되 화면 크기에 맞춰 창 배치와 조작 방식을 조정합니다.

레퍼런스:

- https://posthog.com/
- https://github.com/PostHog/posthog/blob/master/frontend

PostHog는 제품 톤, 정보 밀도, 인터랙션 품질을 참고하는 레퍼런스입니다. PostHog 로고나 브랜드 고유 자산은 사용하지 않습니다.

## MVP 범위

### OS 데스크톱

- OS 스타일 바탕화면을 첫 화면으로 제공합니다.
- 바탕화면에는 핵심 앱 아이콘을 배치합니다.
- 앱 아이콘을 클릭하거나 탭하면 해당 프로그램 창이 열립니다.
- 사용자는 여러 창을 동시에 열 수 있습니다.
- 창은 닫기, 포커스, 앞으로 가져오기를 지원합니다.
- 첫 로드 시 Portfolio 창은 기본으로 자동 오픈합니다.

### Portfolio 앱

- Portfolio 아이콘을 통해 열리는 기본 포트폴리오 프로그램입니다.
- 첫 로드 시 자동으로 열려 사용자의 대표 작업과 소개를 바로 보여줍니다.
- 프로젝트, 역할, 사용 기술, 문제 해결 방식, 결과를 확인할 수 있어야 합니다.
- 다른 앱을 열어도 Portfolio 창은 OS 창 시스템 안에서 함께 관리됩니다.

### Store 앱과 Cart

- Store 아이콘을 통해 미니 쇼핑몰 프로그램을 엽니다.
- 상품 데이터는 MVP에서 정적 데이터로 제공합니다.
- 사용자는 상품 목록을 보고 장바구니에 추가할 수 있습니다.
- 장바구니는 비회원 기준으로 동작합니다.
- 장바구니 아이템, 수량, 합계 금액을 확인할 수 있어야 합니다.
- 장바구니 상태는 `localStorage`에 저장해 새로고침 후에도 유지합니다.
- 실제 결제, 상품 관리자, 백엔드 저장은 후속 확장으로 다룹니다.

### Resume 앱

- Resume 아이콘을 통해 이력서 진입 프로그램을 엽니다.
- MVP에서는 PDF 이력서 또는 이력서 문서로 이동할 수 있는 명확한 진입점을 제공합니다.
- WebView 환경에서도 사용자가 이력서에 접근할 수 있어야 합니다.

### Contact 앱

- Contact 아이콘을 통해 연락 프로그램을 엽니다.
- 이메일, 외부 프로필, 연락 가능한 채널을 제공합니다.
- 메일 링크는 사용자의 기본 메일 클라이언트 또는 브라우저 동작을 따릅니다.
- WebView에서 링크 동작이 막히지 않도록 모바일 셸 구현 시 별도 검증합니다.

### Calculator 앱

- Calculator 아이콘을 통해 미니 계산기 프로그램을 엽니다.
- 기본 사칙연산을 지원합니다.
- 계산 상태는 창 내부에서 관리합니다.
- 작고 독립적인 OS 앱처럼 동작해야 하며 다른 창과 포커스 규칙을 공유합니다.

### 반응형과 WebView

- PC 브라우저에서는 데스크톱 OS에 가까운 창 기반 UI를 제공합니다.
- 모바일 브라우저와 Flutter WebView에서는 같은 앱 아이콘 모델을 유지하되 작은 화면에 맞춰 창을 전체 화면 또는 단순화된 레이아웃으로 표시할 수 있습니다.
- 모든 주요 기능은 PC, 모바일 브라우저, Flutter WebView에서 접근 가능해야 합니다.
- 작은 화면에서 텍스트, 아이콘, 창 헤더, 버튼이 겹치거나 가로 스크롤을 만들지 않아야 합니다.
- 터치 입력으로 앱 열기, 닫기, 주요 버튼 조작이 가능해야 합니다.

### 배포와 모바일 셸

- 웹앱은 Next.js로 구현하고 Vercel에 배포합니다.
- Flutter WebView 앱은 Vercel production URL을 로드합니다.
- Flutter 앱은 로딩, 에러, Android back button 동작을 처리합니다.

## 아이콘과 에셋 정책

- MVP 앱 아이콘은 `lucide-react` 기반의 MIT 라이선스 아이콘을 사용합니다.
- 아이콘은 OS 데스크톱의 프로그램 아이콘처럼 보이도록 일관된 크기, 라벨, 선택 상태를 가집니다.
- PostHog 오픈소스 프론트엔드는 UI/UX 레퍼런스로 참고하되 PostHog 브랜드 고유 자산은 사용하지 않습니다.
- 추후 자체 아이콘 세트가 준비되면 `lucide-react` 아이콘을 교체할 수 있도록 앱 정의와 아이콘 표현을 분리합니다.

## 추후 확장 범위

- 자체 앱 아이콘 세트 제작과 교체.
- 비회원 장바구니 API.
- 세션 기반 장바구니 저장.
- 상품 관리자 페이지.
- 결제 mock.
- 인증 기능.
- 창 위치와 크기 저장.
- 앱 검색 또는 런처.
- 테마 설정.

## 핵심 사용자 흐름

1. 사용자가 웹앱 또는 Flutter WebView 앱을 엽니다.
2. OS 스타일 바탕화면이 표시되고 Portfolio 창이 자동으로 열립니다.
3. 사용자는 Portfolio 창에서 대표 작업과 소개를 확인합니다.
4. 사용자는 바탕화면의 Store 아이콘을 열어 상품을 확인하고 장바구니에 담습니다.
5. 사용자는 Calculator 아이콘을 열어 간단한 계산을 수행합니다.
6. 사용자는 Resume 아이콘을 열어 이력서에 접근합니다.
7. 사용자는 Contact 아이콘을 열어 연락 수단을 확인합니다.
8. 사용자는 열린 창을 닫거나 다른 아이콘을 열며 OS처럼 탐색합니다.

## 완료 기준

- PRD가 OS 스타일 포트폴리오 웹앱이 무엇을 만드는지 설명합니다.
- 핵심 앱이 바탕화면 아이콘에서 열리는 프로그램 창으로 정의되어 있습니다.
- Store/cart, Portfolio, Resume, Contact, Calculator 요구사항이 포함되어 있습니다.
- MVP 범위와 후속 확장 범위가 구분되어 있습니다.
- PC 브라우저, 모바일 브라우저, Flutter WebView 요구사항이 포함되어 있습니다.
- 아이콘은 `lucide-react` 기반으로 시작하고 추후 자체 에셋으로 교체 가능하다는 정책이 명시되어 있습니다.
