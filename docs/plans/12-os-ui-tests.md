# 12. OS UI Tests Plan

## Goal

Test the OS desktop shell behavior.

## Tasks

- Test default Portfolio window visibility.
- Test icon click opens a window.
- Test close button hides a window.
- Test focus behavior if implemented.

## Acceptance Criteria

- OS UI behavior is covered by component or E2E tests.
- Tests pass in `npm run verify`.

## Validation

```bash
cd apps/web
npm run verify
```
