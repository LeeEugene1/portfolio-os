# 개발 하네스

## 목적

이 문서는 로컬 개발 서버 실행, 포트 안내, 완료 후 서버 정리 규칙을 정의합니다.

이슈 작업 흐름과 승인 규칙은 `docs/AGENT_WORKFLOW.md`를 따르고, 웹 검증 명령은 `docs/TEST_PLAN.md`를 따릅니다.

## 로컬 서버 실행

- dev server를 실행하면 사용자에게 실제 포트와 URL을 알려줍니다.
- 예: `http://localhost:3005`
- 요청 포트가 사용 중이면 다음 빈 포트를 사용하고, 어떤 포트로 열었는지 명확히 알립니다.
- 특정 이슈 worktree에서 실행한 서버라면 어떤 worktree 기준인지 함께 확인합니다.

## 포트 확인

3000번대 개발 서버 확인은 아래 명령을 사용합니다.

```bash
lsof -nP -iTCP:3000-3010 -sTCP:LISTEN
```

포트 확인 결과를 사용자에게 전달할 때는 최소한 아래 정보를 포함합니다.

- 열려 있는 포트.
- 해당 포트의 URL.
- 어떤 이슈 또는 worktree용 서버인지 알고 있는 경우 그 용도.

## 포트 선택

- 사용자가 특정 포트를 요청하면 먼저 해당 포트가 비어 있는지 확인합니다.
- 비어 있으면 그 포트로 실행합니다.
- 사용 중이면 사용자에게 알리고 다음 빈 포트를 사용합니다.
- Docker, 시스템 서비스, 다른 이슈 서버가 점유한 포트는 사용자 요청 없이 임의로 종료하지 않습니다.

## 서버 종료

작업 완료 시 관련 로컬 dev server를 종료합니다.

완료로 보는 시점:

- PR 생성 후 사용자가 더 이상 로컬 확인이 필요 없다고 말한 경우.
- PR merge 후 해당 이슈가 닫힌 경우.
- 사용자가 “완료”, “종료”, “정리”처럼 서버 종료 의사를 명확히 말한 경우.

종료 전에는 사용자에게 어떤 포트를 종료하는지 알립니다.

다른 이슈 또는 다른 worktree의 서버는 임의로 종료하지 않습니다.

## 종료 명령

포트를 점유한 프로세스를 확인합니다.

```bash
lsof -nP -iTCP:3000-3010 -sTCP:LISTEN
```

종료가 필요한 서버의 PID만 종료합니다.

```bash
kill <PID>
```

일반 종료가 실패하고 사용자가 해당 서버 종료를 요청한 경우에만 강제 종료를 사용합니다.

```bash
kill -9 <PID>
```

## 최종 보고

로컬 서버를 실행한 작업의 최종 응답에는 아래 중 하나를 포함합니다.

- 계속 실행 중인 서버 URL.
- 종료한 서버 포트.
- 종료하지 않은 이유.

예:

```txt
로컬 서버는 http://localhost:3005 에서 실행 중입니다.
```

```txt
작업 완료 후 관련 dev server 3005번을 종료했습니다.
```
